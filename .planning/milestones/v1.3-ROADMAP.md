# Milestone v1.3: Visual Overhaul

**Status:** SHIPPED 2026-01-20
**Phases:** 17-20
**Total Plans:** 13

## Overview

Transform ASTN from generic shadcn/ui (3.7/10) to a warm, memorable visual identity (8+/10). The "Warm & Human" aesthetic communicates that AI safety is about people. Build order: Foundation/Tokens first (everything depends on them), then Visual Polish (apply tokens consistently), then Motion (requires stable layout), then Polish/Integration (reveals edge cases).

## Milestone Summary

**Key Decisions:**
- OKLCH color space for perceptual uniformity
- Lora for headings, Plus Jakarta Sans for body
- Spring easing (1.56 overshoot) for organic motion
- Stagger cap at 9 items (450ms max delay)
- Dark mode primary stays coral for brand consistency
- Cookie-based SSR theme detection for no-flash experience
- View Transitions API for native page animation

**Issues Resolved:**
- FOIT/FOUT eliminated via woff2 preloads
- Dark mode flash eliminated via SSR cookie detection
- Scrollbar layout shift prevented with scrollbar-gutter: stable

**Technical Debt Incurred:**
- 8+ routes still use bg-slate-50 instead of GradientBg (settings, attendance, org admin)
- 35+ headings use font-bold instead of font-display font-semibold
- Unused animation keyframes (animate-fade-in, etc.) in favor of tw-animate-css
