**coursePrompts table:**

- `programId: v.id('programs')`
- `moduleId: v.optional(v.id('programModules'))` -- denormalized for indexing
- `sessionId: v.optional(v.id('programSessions'))` -- denormalized for indexing
- `attachedTo: v.union(...)` -- discriminated union with two variants:
  - `{ type: 'module', moduleId: v.id('programModules') }`
  - `{ type: 'session_phase', sessionId: v.id('programSessions'), phaseIndex: v.number() }`
- `title: v.string()`
- `body: v.optional(v.string())` -- markdown content
- `orderIndex: v.number()`
- `fields: v.array(v.object({ id: v.string(), type: v.union(v.literal('text'), v.literal('choice'), v.literal('multiple_choice')), label: v.string(), required: v.boolean(), placeholder: v.optional(v.string()), options: v.optional(v.array(v.object({ id: v.string(), label: v.string() }))), maxLength: v.optional(v.number()) }))`
- `revealMode: v.union(v.literal('immediate'), v.literal('facilitator_only'), v.literal('write_then_reveal'))`
- `revealedAt: v.optional(v.number())` -- timestamp when reveal triggered (LOCKED DECISION from STATE.md)
- `createdBy: v.string()` -- Clerk userId
- `createdAt: v.number()`
- `updatedAt: v.number()`
- Indexes: `by_program` on `['programId']`, `by_module` on `['moduleId']`, `by_session` on `['sessionId']`, `by_program_and_order` on `['programId', 'orderIndex']`

**coursePromptResponses table:**

- `promptId: v.id('coursePrompts')`
- `programId: v.id('programs')` -- denormalized
- `userId: v.string()` -- Clerk userId
- `fieldResponses: v.array(v.object({ fieldId: v.string(), textValue: v.optional(v.string()), selectedOptionIds: v.optional(v.array(v.string())) }))`
- `status: v.union(v.literal('draft'), v.literal('submitted'))`
- `spotlighted: v.optional(v.boolean())`
- `spotlightedBy: v.optional(v.string())`
- `spotlightedAt: v.optional(v.number())`
- `savedAt: v.number()`
- `submittedAt: v.optional(v.number())`
- Indexes: `by_prompt` on `['promptId']`, `by_prompt_and_user` on `['promptId', 'userId']`, `by_program_and_user` on `['programId', 'userId']`

Follow existing schema patterns -- use `v.optional()` for nullable fields, descriptive index names per convex/CLAUDE.md.

<task type="auto">
  <name>Task 2: Create backend functions for prompts and responses</name>
  <files>convex/course/prompts.ts, convex/course/responses.ts</files>
  <action>
Create the `convex/course/` directory with two files.

**convex/course/prompts.ts** -- Prompt CRUD + reveal:

1. Import helpers: Copy the `requireOrgAdmin` pattern from `convex/programs.ts` (lines 9-27). Also import `getUserId` from `../lib/auth`. Import `mutation, query` from `../_generated/server`, `v, ConvexError` from `convex/values`.

2. Define shared validators at module level:
   - `fieldValidator` for the fields array element
   - `attachedToValidator` for the discriminated union
   - `revealModeValidator` for the three literal modes

3. `create` mutation: Args include programId, attachedTo, title, body (optional), orderIndex, fields array, revealMode. Requires orgAdmin of the program's org. Computes denormalized `moduleId`/`sessionId` from attachedTo. Sets createdBy, createdAt, updatedAt. Returns `v.id('coursePrompts')`.

4. `update` mutation: Args include promptId plus optional title, body, orderIndex, fields, revealMode. Requires orgAdmin. Only allows revealMode change if no submitted responses exist yet (to prevent confusion). Patches the document. Returns `v.null()`.

5. `remove` mutation: Args promptId. Requires orgAdmin. Deletes the prompt AND all its responses (query by_prompt index, loop delete). Returns `v.null()`.

6. `get` query: Args promptId. Returns the prompt document or null. Uses `checkProgramAccess` pattern -- return null if no access. Include full return validator.

7. `getByModule` query: Args moduleId. Returns all prompts for that module ordered by orderIndex. Uses `by_module` index. Requires program access (look up module to get programId, then check access). Return `v.array(...)`.

8. `getBySession` query: Args sessionId. Same pattern as getByModule but uses `by_session` index.

9. `triggerReveal` mutation: Args promptId. Requires orgAdmin. Validates prompt is `write_then_reveal` mode. Sets `revealedAt: Date.now()` and `updatedAt`. Idempotent -- if already revealed, return null. Returns `v.null()`.

**convex/course/responses.ts** -- Response save + visibility queries + spotlight:

1. `saveResponse` mutation: Args promptId, fieldResponses array, submit boolean. Gets userId via requireAuth. Looks up prompt to get programId. Checks program access (enrolled/completed or admin). Upserts via `by_prompt_and_user` index -- if existing draft, patch it; if existing submitted and trying to save again, throw ConvexError('Response already submitted'). On new: insert with status draft/submitted, savedAt, submittedAt. Returns `v.id('coursePromptResponses')`.

2. `getMyResponse` query: Args promptId. Returns the current user's response (draft or submitted) or null. Uses `by_prompt_and_user` index.

3. `getPromptResponses` query: Args promptId. This is the CRITICAL visibility-filtered query. Implementation:
   - Load the prompt to get revealMode and revealedAt
   - Check program access to determine if admin
   - Load all responses via `by_prompt` index
   - If admin: return all responses
   - If participant with `immediate` mode: return all submitted responses
   - If participant with `facilitator_only` mode: return only own response
   - If participant with `write_then_reveal` mode: if `prompt.revealedAt` is set, return all submitted + spotlighted; otherwise return only own response
   - Always include the user's own response regardless of mode
   - Return validator must include all response fields plus `_id` and `_creationTime`

4. `toggleSpotlight` mutation: Args responseId. Requires orgAdmin of the program. Toggles `spotlighted` boolean, sets/clears `spotlightedBy` and `spotlightedAt`. Returns `v.null()`.

CRITICAL RULES (from convex/CLAUDE.md):

- Every function MUST have both `args:` and `returns:` validators
- NEVER use `.filter()` -- use `.withIndex()` only
- Use `ConvexError` for user-facing errors
- The `checkProgramAccess` and `requireOrgAdmin` helpers must be copied/imported since they are NOT exported from programs.ts (they are file-local functions). Recreate them in the course files or create a shared helper in `convex/course/_helpers.ts`.
