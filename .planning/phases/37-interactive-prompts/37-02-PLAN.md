<task type="auto">
  <name>Task 2: Create PromptRenderer with draft save and submit flow</name>
  <files>src/components/course/PromptRenderer.tsx</files>
  <action>
Create the main `PromptRenderer` component -- the single reusable primitive for the entire prompt system.

**Interface:**

```
interface PromptRendererProps {
  promptId: Id<'coursePrompts'>
  mode: 'participate' | 'review'  // participant responding vs facilitator viewing
}
```

**Data fetching (Convex reactive queries):**

- `useQuery(api.course.prompts.get, { promptId })` -- the prompt document
- `useQuery(api.course.responses.getMyResponse, { promptId })` -- current user's response (participate mode)
- `useQuery(api.course.responses.getPromptResponses, mode === 'review' ? { promptId } : 'skip')` -- all responses (review mode only)
- `useMutation(api.course.responses.saveResponse)` -- for saving/submitting

**State management:**

- Local state: `fieldValues` -- a `Record<string, { textValue?: string, selectedOptionIds?: string[] }>` initialized from `myResponse.fieldResponses` when data loads
- Track `isDirty` (local changes not yet saved) and `isSubmitted` (response has been submitted)
- Track `isSaving` for loading state

**Participate mode rendering:**

1. Render `PromptMarkdownBody` with prompt body (if exists)
2. Map over `prompt.fields` and render appropriate field component:
   - type 'text' -> `PromptFieldText`
   - type 'choice' -> `PromptFieldChoice`
   - type 'multiple_choice' -> `PromptFieldMultiChoice`
3. Pass `disabled={isSubmitted}` to all fields (no editing after submission)
4. Show two buttons at bottom:
   - "Save Draft" -- calls `saveResponse({ promptId, fieldResponses: buildFieldResponses(), submit: false })`, shows toast.success('Draft saved')
   - "Submit" -- calls `saveResponse({ ..., submit: true })`, shows toast.success('Response submitted'). Disable after submission.
5. Show status indicator: "Draft saved at {time}" or "Submitted at {time}"
6. Initialize `fieldValues` from `myResponse` when it loads (useEffect)

**Review mode rendering:**

1. Render prompt body
2. Show response count ("12 responses")
3. List all responses from `allResponses`:
   - For each response, show respondent info and their field values
   - Show `SpotlightBadge` on spotlighted responses
4. This is a read-only view -- no input fields

**Visibility awareness:**

- In participate mode, if `prompt.revealMode === 'write_then_reveal'` and `!prompt.revealedAt`:
  - Show info banner: "Your response will be visible to others after the facilitator reveals all responses"
- If `prompt.revealMode === 'facilitator_only'`:
  - Show info banner: "Only the facilitator can see responses"
- After reveal (revealedAt is set), show other submitted responses below own response

**Error handling:**

- Use try/catch around save/submit mutations
- Show toast.error on failure
- Use `isSaving` state for button loading indicators

**Loading/empty states:**

- Show skeleton while prompt data loads
- Show "No prompt found" if prompt is null

Wrap in a shadcn Card component for consistent styling. Use Tailwind classes matching project conventions.
</action>
