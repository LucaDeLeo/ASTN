Create `PromptForm.tsx` -- the facilitator form for creating and editing prompts.

**Interface:**

```
interface PromptFormProps {
  programId: Id<'programs'>
  attachedTo: { type: 'module', moduleId: Id<'programModules'> } | { type: 'session_phase', sessionId: Id<'programSessions'>, phaseIndex: number }
  orderIndex: number
  existingPrompt?: Doc<'coursePrompts'>  // If provided, editing mode
  onSuccess?: () => void  // Called after successful create/update
  onCancel?: () => void
}
```

**Form state (local React state):**

- `title: string`
- `body: string` (markdown)
- `revealMode: 'immediate' | 'facilitator_only' | 'write_then_reveal'`
- `fields: Array<{ id: string, type: 'text' | 'choice' | 'multiple_choice', label: string, required: boolean, placeholder: string, options: Array<{ id: string, label: string }>, maxLength: number | undefined }>`
- `isSaving: boolean`

Initialize from `existingPrompt` if provided (edit mode).

**Form layout:**

1. **Title** -- shadcn `Input` with label "Prompt Title"
2. **Body** -- shadcn `Textarea` with label "Instructions (Markdown)" and placeholder "You can use **bold**, _italic_, lists, and links..."
3. **Reveal Mode** -- shadcn `Select` with three options:
   - "Immediate" -- "Responses visible to everyone right away"
   - "Facilitator Only" -- "Only you can see responses"
   - "Write Then Reveal" -- "Hidden until you trigger reveal"
4. **Fields section** -- "Questions" heading with "Add Field" button
   - Each field renders as a card with:
     - Field type selector (Select: Text, Single Choice, Multiple Choice)
     - Label input (Input: "Question text")
     - Required checkbox (Checkbox)
     - Placeholder input (Input, optional, only for text type)
     - Max length input (Input, optional, only for text type, number)
     - Options builder (only for choice/multiple_choice):
       - List of option labels with delete button per option
       - "Add Option" button at bottom
     - Delete field button (Trash2 icon) and move up/down buttons (ArrowUp/ArrowDown icons)
   - Generate stable field IDs: `field_${index}` for new fields, preserve existing IDs on edit

5. **Action buttons:**
   - "Create Prompt" / "Update Prompt" -- calls create or update mutation
   - "Cancel" -- calls onCancel

**Field builder interactions:**

- "Add Field" button: appends a new field with type 'text', empty label, required: false
- Delete field: removes from array
- Move up/down: swaps array positions (simple reorder)
- Changing field type: clears options if switching away from choice/multiple_choice, initializes empty options array if switching to choice/multiple_choice

**Submit handler:**

- Validate: title required, at least one field, all fields have labels, choice/multiple_choice fields have at least 2 options
- Call `api.course.prompts.create` or `api.course.prompts.update` mutation
- Show toast.success on success, toast.error on failure
- Call onSuccess callback

**Styling:** Use shadcn Card for the form container, consistent spacing with `space-y-4`, field cards with border and padding.
</action>

**PromptResponseViewer.tsx** -- Facilitator view of all responses for a prompt:

**Interface:**

```
interface PromptResponseViewerProps {
  promptId: Id<'coursePrompts'>
}
```

**Data fetching:**

- `useQuery(api.course.prompts.get, { promptId })` -- prompt document
- `useQuery(api.course.responses.getPromptResponses, { promptId })` -- all responses (admin sees all)
- `useMutation(api.course.responses.toggleSpotlight)` -- spotlight toggle

**Rendering:**

1. Header: prompt title + response count ("8 of 10 participants responded")
2. Include `PromptRevealControl` if prompt is write_then_reveal mode
3. Response list -- for each response:
   - Participant identifier (userId -- show as truncated clerk ID for now; future phases will resolve to names)
   - Status badge: "Draft" (muted) or "Submitted" (green)
   - For each field in the prompt, show the field label and the participant's answer:
     - Text fields: show the text value
     - Choice fields: show the selected option label
     - Multiple choice: show all selected option labels as badges
   - Spotlight toggle button (Star icon): click calls `toggleSpotlight({ responseId })`
   - If spotlighted, show `SpotlightBadge` component
   - Submitted timestamp
4. Empty state: "No responses yet" with muted styling

**Styling:** Use shadcn Card for each response. Alternate background colors or add separator lines between responses for readability.

**PromptRevealControl.tsx** -- Reveal trigger for write_then_reveal prompts:

**Interface:**

```
interface PromptRevealControlProps {
  promptId: Id<'coursePrompts'>
  revealMode: 'immediate' | 'facilitator_only' | 'write_then_reveal'
  revealedAt?: number
}
```

**Rendering:**

- Only render UI for `write_then_reveal` mode
- If NOT yet revealed (`!revealedAt`):
  - Show an alert/banner: "Responses are currently hidden from participants"
  - Show "Reveal All Responses" button with Eye icon
  - On click: confirm dialog ("Are you sure? This cannot be undone.") then call `api.course.prompts.triggerReveal({ promptId })`
  - Show toast.success('Responses revealed to all participants')
- If already revealed:
  - Show info text: "Responses revealed at {formatted time}" with CheckCircle icon
  - No action button

**Styling:** Use shadcn `Alert` or a styled div for the banner. Use shadcn `AlertDialog` for the confirmation.

Import `SpotlightBadge` from `./SpotlightBadge` in the response viewer. Use `lucide-react` icons: `Star`, `Eye`, `CheckCircle2`, `Clock`.
</action>
