**1. Add `courseSidebarThreads` table to `convex/schema.ts`:**

Add after the `coursePromptResponses` table definition:

```typescript
// Course sidebar threads (Phase 38) - maps user+module to @convex-dev/agent thread
courseSidebarThreads: defineTable({
  userId: v.string(),
  moduleId: v.id('programModules'),
  programId: v.id('programs'), // Denormalized for facilitator queries
  threadId: v.string(), // @convex-dev/agent thread ID
  createdAt: v.number(),
})
  .index('by_userId_and_moduleId', ['userId', 'moduleId'])
  .index('by_programId', ['programId'])
  .index('by_programId_and_userId', ['programId', 'userId']),
```

**2. Also add `aiFeedback` field to the existing `coursePrompts` table schema:**

Add `aiFeedback: v.optional(v.boolean()),` after the `revealedAt` field in the `coursePrompts` table definition. This is needed for SIDE-04 proactive feedback (Plan 02 will wire the trigger).

**3. Create `convex/course/sidebarAgent.ts`:**

This file needs `'use node'` at the top. Follow the exact pattern from `convex/agent/index.ts` and `convex/agent/actions.ts`.

- Define `learningAgent` using `new Agent(components.agent, {...})` with:
  - `name: 'learning-partner'`
  - `languageModel: anthropic.chat('claude-sonnet-4-6')`
  - `instructions: ''` (dynamic per-turn)
  - `tools: {}` (no tools needed -- learning partner is conversational only)
  - `stopWhen: stepCountIs(3)`

- Create `streamResponse` as an `internalAction` with args: `threadId` (string), `promptMessageId` (string), `moduleId` (Id<'programModules'>), `userId` (string). Handler:
  1. Call `ctx.runQuery(internal.course.sidebar.buildModuleContext, { moduleId, userId })` to get context
  2. Build system prompt via `buildLearningSystemPrompt(context)` (defined in same file)
  3. Call `learningAgent.continueThread(ctx, { threadId })` then `thread.streamText({ promptMessageId, system }, { saveStreamDeltas: { chunking: 'word', throttleMs: 100 } })`
  4. `await result.consumeStream()` then `return null`
  5. Use the same type assertion pattern as `convex/agent/actions.ts` line 163: `{ promptMessageId, system } as Parameters<typeof thread.streamText>[0]`

- Create `streamFeedback` as a separate `internalAction` (same signature + `promptId: v.id('coursePrompts')`). Similar to `streamResponse` but the context builder also includes the specific prompt response the user just submitted. This will be called by the proactive feedback trigger in Plan 02.

- Create `buildLearningSystemPrompt(context)` function (not exported to Convex -- just a local helper). The system prompt must:
  - Identify the AI as a learning partner for the AI safety course module
  - Include module title, description, materials list (title + type + completion status, NOT full content)
  - Include participant progress (completed/total materials)
  - Include participant's prompt responses (title + truncated text responses, max ~500 chars each)
  - Enforce Socratic method: "When the participant asks about exercises or course concepts, respond with guiding questions that help them think through the problem. Never give direct answers. Help them discover insights themselves."
  - Include study prioritization guidance: "When asked about study priorities or time management, consider which materials are incomplete, which are marked essential, and proximity to the next session date."
  - Include the module's session date context if available
  - Keep total prompt under ~2000 tokens (material titles/status only, not full URLs or descriptions)

**4. Create `convex/course/sidebar.ts`:**

Follow the pattern from `convex/agent/threadOps.ts` exactly.

- `getOrCreateThread` mutation: args `{ moduleId: v.id('programModules') }`, returns `v.string()`. Handler:
  1. `requireAuth(ctx)` to get userId
  2. Query `courseSidebarThreads` by `by_userId_and_moduleId` index
  3. If exists, return `existing.threadId`
  4. If not, verify module exists, get program, call `checkProgramAccess(ctx, program)` to confirm access
  5. `createThread(ctx, components.agent, { userId })` to get threadId
  6. Insert into `courseSidebarThreads` with userId, moduleId, programId, threadId, createdAt
  7. Return threadId

- `sendMessage` mutation: args `{ threadId: v.string(), prompt: v.string(), moduleId: v.id('programModules') }`, returns `v.string()`. Handler:
  1. `requireAuth(ctx)` to get userId
  2. `saveMessage(ctx, components.agent, { threadId, prompt })` to get messageId
  3. `ctx.scheduler.runAfter(0, internal.course.sidebarAgent.streamResponse, { threadId, promptMessageId: messageId, moduleId, userId })`
  4. Return messageId

- `abortGeneration` mutation: args `{ threadId: v.string() }`, returns `v.number()`. Same pattern as `convex/agent/threadOps.ts` `abortGeneration`.

- `buildModuleContext` as `internalQuery`: args `{ moduleId: v.id('programModules'), userId: v.string() }`, returns `v.any()`. Handler:
  1. Get module from DB
  2. Get program from DB
  3. Get all materials for this module via `programModules` (the module has materials array or query materials)
  4. Get materialProgress for this user+module via `by_module_and_user` index
  5. Get coursePrompts for this module via `by_moduleId` index
  6. For each prompt, get user's coursePromptResponse via `by_promptId_and_userId` index
  7. Get next session date: query `programSessions` by `by_program` index, find first with date > now
  8. Return structured object: `{ moduleTitle, moduleDescription, materials: [{title, type, completed}], progress: {completed, total}, prompts: [{title, userResponse}], nextSessionDate }`

Import `requireAuth` from `../lib/auth`, `checkProgramAccess` from `./_helpers`, `createThread` and `saveMessage` and `abortStream` and `listStreams` from `@convex-dev/agent`, `components` and `internal` from `../_generated/api`.
</action>
