**2. Wire proactive feedback trigger in `convex/course/responses.ts`:**

In the `saveResponse` mutation, after successful insert/patch when `submit === true`:

```typescript
// Proactive AI feedback (SIDE-04)
if (submit) {
  const prompt = await ctx.db.get('coursePrompts', promptId)
  // aiFeedback defaults to true when undefined
  if (prompt && prompt.aiFeedback !== false) {
    const module = prompt.moduleId
      ? await ctx.db.get('programModules', prompt.moduleId)
      : null
    if (module) {
      // Find or note that thread may not exist yet
      const existingThread = await ctx.db
        .query('courseSidebarThreads')
        .withIndex('by_userId_and_moduleId', (q) =>
          q.eq('userId', userId).eq('moduleId', module._id),
        )
        .first()

      if (existingThread) {
        // Inject a system message describing the submission
        const { messageId } = await saveMessage(ctx, components.agent, {
          threadId: existingThread.threadId,
          prompt: `[System: I just submitted my response to the exercise "${prompt.title}". Can you review it and give me feedback?]`,
        })

        await ctx.scheduler.runAfter(
          0,
          internal.course.sidebarAgent.streamFeedback,
          {
            threadId: existingThread.threadId,
            promptMessageId: messageId,
            moduleId: module._id,
            userId,
            promptId,
          },
        )
      }
      // If no thread exists, skip proactive feedback -- it will be available
      // when the user opens the sidebar and the AI can see their responses
    }
  }
}
```

Note: The `prompt` variable is already fetched earlier in the handler (line ~38). Reuse that reference instead of re-fetching. Import `saveMessage` from `@convex-dev/agent`, `components` and `internal` from `../_generated/api`.

The proactive feedback only fires if: (a) the response is being submitted (not draft-saved), (b) the prompt has `aiFeedback !== false` (defaults to true), (c) the prompt is attached to a module (not a session phase), and (d) the user already has a sidebar thread for that module. This avoids creating threads for users who never opened the sidebar.
