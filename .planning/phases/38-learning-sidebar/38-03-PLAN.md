**1. Create `src/components/course/FacilitatorConversations.tsx`:**

Props: `{ programId: Id<'programs'> }`

Component structure:

- Use `useQuery(api.course.sidebarQueries.getParticipantThreads, { programId })` to get all threads
- Group threads by userId to build a participant list (each participant may have threads for multiple modules)
- Left panel: List of participants with names, showing count of module conversations (e.g., "3 modules")
- When a participant is selected (useState), show their module threads via `useQuery(api.course.sidebarQueries.getParticipantThreadsByUser, { programId, userId: selectedUserId })`
- Right panel: List of module threads for the selected participant. Each shows module name and creation date. Clicking a thread opens `ConversationViewer`.
- Use a `useState<string | null>` for `selectedThreadId` to control which conversation is being viewed
- When a thread is selected, render `<ConversationViewer threadId={selectedThreadId} />` in a Dialog or inline expanding panel
- If no threads exist, show an empty state: "No AI sidebar conversations yet. Conversations will appear here once participants start using the AI learning partner."
- Use `Card` components for layout. Use `Badge` for module names. Use `Users` icon from lucide-react for the header.

**2. Create `src/components/course/ConversationViewer.tsx`:**

Props: `{ threadId: string }`

Component structure:

- Use `useUIMessages(api.course.sidebarQueries.listMessages, { threadId }, { initialNumItems: 100, stream: false })` -- note `stream: false` since this is a read-only view, not a live chat
- Render messages in a scrollable container
- User messages: right-aligned with primary bg (same styling as the sidebar chat)
- Assistant messages: left-aligned with muted bg, render markdown via `react-markdown` + `remark-gfm`
- Show message timestamps using `date-fns` `formatDistanceToNow` or similar
- Read-only: no input area, no send button
- Header: "Conversation" with module name if available
- If no messages, show "No messages in this conversation."
