```typescript
// Agent proposals (Phase 39) â€” facilitator agent propose-and-approve workflow
agentProposals: defineTable({
  programId: v.id('programs'),
  type: v.union(
    v.literal('comment'),
    v.literal('message'),
    v.literal('pairs'),
    v.literal('summary'),
    v.literal('flag'),
    v.literal('prompt'),
  ),
  targetId: v.optional(v.string()), // e.g., coursePromptResponses ID for comments, sessionId for summaries
  targetType: v.optional(v.string()), // e.g., "promptResponse", "session", "participant"
  content: v.string(), // The proposed text/action
  status: v.union(
    v.literal('proposed'),
    v.literal('approved'),
    v.literal('edited'),
    v.literal('dismissed'),
  ),
  editedContent: v.optional(v.string()), // Content after facilitator edit
  approvedBy: v.optional(v.string()), // Clerk userId of approver
  approvedAt: v.optional(v.number()),
  createdAt: v.number(),
})
  .index('by_programId', ['programId'])
  .index('by_programId_and_status', ['programId', 'status'])
  .index('by_targetId', ['targetId']),

// Facilitator comments (Phase 39) â€” approved feedback on participant responses
facilitatorComments: defineTable({
  promptResponseId: v.id('coursePromptResponses'),
  programId: v.id('programs'),
  authorId: v.string(), // Clerk userId â€” facilitator who wrote/approved
  content: v.string(),
  fromAgent: v.boolean(), // Was this proposed by the agent?
  createdAt: v.number(),
})
  .index('by_promptResponseId', ['promptResponseId'])
  .index('by_programId', ['programId'])
  .index('by_programId_and_authorId', ['programId', 'authorId']),
```

Use the same `v` import and `defineTable` pattern as other course tables. Ensure indexes match the query patterns in subsequent tasks.

**Create `convex/course/proposals.ts`:**

This file handles the full proposal lifecycle. Import `requireOrgAdmin` from `./_helpers`.

**1. `createProposal` mutation** (internal â€” only called by agent tools, not frontend):

```typescript
export const createProposal = internalMutation({
  args: {
    programId: v.id('programs'),
    type: v.union(
      v.literal('comment'),
      v.literal('message'),
      v.literal('pairs'),
      v.literal('summary'),
      v.literal('flag'),
      v.literal('prompt'),
    ),
    targetId: v.optional(v.string()),
    targetType: v.optional(v.string()),
    content: v.string(),
  },
  returns: v.id('agentProposals'),
  handler: async (ctx, args) => {
    return await ctx.db.insert('agentProposals', {
      ...args,
      status: 'proposed',
      createdAt: Date.now(),
    })
  },
})
```

Import `internalMutation` from `../_generated/server`.

**2. `approveProposal` mutation** (public â€” facilitator clicks approve):

Args: `{ proposalId: v.id('agentProposals') }`. Handler:

1. Get proposal from DB, throw if not found or not `status: 'proposed'`
2. Get program from DB, call `requireOrgAdmin(ctx, program.orgId)`
3. Get userId via `requireAuth(ctx)`
4. Patch proposal: `status: 'approved'`, `approvedBy: userId`, `approvedAt: Date.now()`
5. **If type is 'comment' and targetId is set:** Create a `facilitatorComments` record:
   - `promptResponseId: proposal.targetId as Id<'coursePromptResponses'>`
   - `programId: proposal.programId`
   - `authorId: userId`
   - `content: proposal.content`
   - `fromAgent: true`
   - `createdAt: Date.now()`
6. Return `{ success: true }`

**3. `editAndApproveProposal` mutation** (public):

Args: `{ proposalId: v.id('agentProposals'), editedContent: v.string() }`. Handler:

1. Same auth checks as approveProposal
2. Patch proposal: `status: 'edited'`, `editedContent`, `approvedBy: userId`, `approvedAt: Date.now()`
3. **If type is 'comment' and targetId is set:** Create `facilitatorComments` with `content: editedContent` and `fromAgent: true`
4. Return `{ success: true }`

**4. `dismissProposal` mutation** (public):

Args: `{ proposalId: v.id('agentProposals') }`. Handler:

1. Get proposal, verify status is 'proposed'
2. Auth check via requireOrgAdmin
3. Patch proposal: `status: 'dismissed'`
4. Return `{ success: true }`

**5. `getProposalsByProgram` query** (public):

Args: `{ programId: v.id('programs'), status: v.optional(v.string()) }`. Handler:

1. Get program, `requireOrgAdmin(ctx, program.orgId)`
2. Query `agentProposals` by `by_programId_and_status` if status provided, otherwise `by_programId`
3. Return proposals array sorted by `createdAt` descending

**6. `getProposalsByTarget` query** (public):

Args: `{ targetId: v.string() }`. Handler:

1. `requireAuth(ctx)` â€” admin check happens via the parent page context
2. Query `agentProposals` by `by_targetId` index
3. Filter to `status: 'proposed'` only (dismissed/approved are historical)
4. Return proposals array

Import `v` from `convex/values`, `mutation`, `query`, `internalMutation` from `../_generated/server`, `requireOrgAdmin` from `./_helpers`, `requireAuth` from `../lib/auth`.

**Create `convex/course/facilitatorQueries.ts`:**

These queries provide the data the facilitator agent's read tools will consume. All require org admin.

**1. `getParticipantProgress` query:**

Args: `{ programId: v.id('programs') }`. Handler:

1. Get program, `requireOrgAdmin(ctx, program.orgId)`
2. Get all participants via `programParticipation` `by_program` index, filter `status: 'enrolled'`
3. Get all modules for the program via `programModules` `by_program` index
4. For each participant:
   a. Count completed materials via `materialProgress` `by_program_and_user` index
   b. Count total materials across all modules (sum of `module.materials.length`)
   c. Count submitted responses via `coursePromptResponses` `by_programId_and_userId` index, filter `status: 'submitted'`
   d. Count total prompts via `coursePrompts` `by_programId` index
   e. Look up profile name via `profiles` `by_user` index
5. Return array of: `{ userId, name, materialsCompleted, materialsTotal, promptsSubmitted, promptsTotal }`

**2. `getResponseCounts` query:**

Args: `{ programId: v.id('programs') }`. Handler:

1. Get program, `requireOrgAdmin(ctx, program.orgId)`
2. Get all prompts for program via `coursePrompts` `by_programId` index
3. For each prompt, count responses via `coursePromptResponses` `by_promptId` index, filter `status: 'submitted'`
4. Get enrolled participant count for denominator
5. Return array of: `{ promptId, promptTitle, moduleId, responseCount, participantCount }`

**3. `getAttendanceSummary` query:**

Args: `{ programId: v.id('programs') }`. Handler:

1. Get program, `requireOrgAdmin(ctx, program.orgId)`
2. Get all sessions via `programSessions` `by_program` index
3. For each session, count attendance records via `sessionAttendance` `by_session` index
4. Get enrolled participant count for denominator
5. Return array of: `{ sessionId, sessionTitle, dayNumber, date, attendeeCount, participantCount }`

All queries must have `args` and `returns` validators. Use `v.any()` for returns (consistent with existing pattern in sidebarQueries.ts).

Import `requireOrgAdmin` from `./_helpers`, `query` from `../_generated/server`.
</action>
