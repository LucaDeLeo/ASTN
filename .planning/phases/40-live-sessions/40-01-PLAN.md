<task type="auto">
  <name>Task 1: Add session runner schema tables and migrate coursePrompts attachedTo</name>
  <files>convex/schema.ts</files>
  <action>
Add five new tables to convex/schema.ts after the existing coursePromptResponses table:

1. **sessionPhases** — ordered agenda items within a session:
   - sessionId: v.id('programSessions')
   - programId: v.id('programs') — denormalized
   - title: v.string()
   - durationMs: v.number() — default duration in milliseconds
   - notes: v.optional(v.string()) — facilitator-only notes
   - promptIds: v.optional(v.array(v.id('coursePrompts'))) — attached prompts
   - pairConfig: v.optional(v.object({ strategy: v.union(v.literal('random'), v.literal('complementary'), v.literal('manual')), sourcePromptId: v.optional(v.id('coursePrompts')), sourceFieldId: v.optional(v.string()) }))
   - orderIndex: v.number()
   - createdAt: v.number()
   - updatedAt: v.number()
   - Indexes: by_sessionId ['sessionId'], by_sessionId_and_order ['sessionId', 'orderIndex']

2. **sessionLiveState** — single document per live session (hot state):
   - sessionId: v.id('programSessions')
   - programId: v.id('programs') — denormalized
   - status: v.union(v.literal('running'), v.literal('paused'), v.literal('completed'))
   - currentPhaseId: v.id('sessionPhases')
   - phaseStartedAt: v.number()
   - phaseDurationMs: v.number()
   - activePromptIds: v.array(v.id('coursePrompts'))
   - startedAt: v.number()
   - startedBy: v.string() — Clerk userId
   - completedAt: v.optional(v.number())
   - Indexes: by_sessionId ['sessionId'], by_programId_and_status ['programId', 'status']

3. **sessionPresence** — typing/activity indicators (separate from responses per CONTEXT.md decision):
   - sessionId: v.id('programSessions')
   - userId: v.string() — Clerk userId
   - phaseId: v.id('sessionPhases')
   - status: v.union(v.literal('typing'), v.literal('idle'), v.literal('submitted'))
   - lastSeen: v.number()
   - Indexes: by_sessionId ['sessionId'], by_sessionId_and_userId ['sessionId', 'userId'], by_sessionId_and_phaseId ['sessionId', 'phaseId']

4. **sessionPairAssignments** — pair/trio assignments per phase:
   - sessionId: v.id('programSessions')
   - phaseId: v.id('sessionPhases')
   - programId: v.id('programs') — denormalized
   - strategy: v.union(v.literal('random'), v.literal('complementary'), v.literal('manual'))
   - pairs: v.array(v.object({ members: v.array(v.string()) })) — Clerk userIds, 2 or 3 for trios
   - createdAt: v.number()
   - createdBy: v.string() — Clerk userId
   - Indexes: by_sessionId ['sessionId'], by_sessionId_and_phaseId ['sessionId', 'phaseId']

5. **sessionPhaseResults** — actual durations recorded after each phase:
   - sessionId: v.id('programSessions')
   - phaseId: v.id('sessionPhases')
   - actualDurationMs: v.number()
   - startedAt: v.number()
   - endedAt: v.number()
   - Indexes: by_sessionId ['sessionId']

Also migrate the coursePrompts.attachedTo.session_phase variant from `phaseIndex: v.number()` to `phaseId: v.id('sessionPhases')`. This is safe because no session-phase prompts exist yet (session phases are being created in this phase). Keep the denormalized `sessionId` field.

IMPORTANT: Follow convex/CLAUDE.md rules — index names must include all indexed fields (by_field1_and_field2 format). Use string for all user IDs (Clerk IDs per CONTEXT.md decision).
</action>

<task type="auto">
  <name>Task 2: Session phase CRUD mutations and base queries</name>
  <files>convex/course/sessionSetup.ts, convex/course/sessionQueries.ts</files>
  <action>
Create **convex/course/sessionSetup.ts** with four mutations for phase CRUD. All mutations require org admin auth via `requireOrgAdmin` from `./_helpers`. All mutations check for an active live session (`sessionLiveState` with status 'running' for this session) and throw if found (locked during live per CONTEXT.md decision).

1. **createPhase** mutation:
   - Args: sessionId, title, durationMs, notes (optional), promptIds (optional), pairConfig (optional)
   - Look up session -> program -> requireOrgAdmin
   - Check no running live state for this session
   - Compute orderIndex as max existing orderIndex + 1 (or 0 if no phases)
   - Insert into sessionPhases with programId denormalized, createdAt/updatedAt
   - Returns: v.id('sessionPhases')

2. **updatePhase** mutation:
   - Args: phaseId, partial fields (title, durationMs, notes, promptIds, pairConfig — all optional)
   - Look up phase -> session -> program -> requireOrgAdmin
   - Check no running live state
   - Patch the phase, set updatedAt
   - Returns: v.null()

3. **reorderPhases** mutation:
   - Args: sessionId, phaseIds (array of Id<'sessionPhases'> in desired order)
   - Look up session -> program -> requireOrgAdmin
   - Check no running live state
   - For each phaseId in array, patch its orderIndex to the array index
   - Returns: v.null()

4. **deletePhase** mutation:
   - Args: phaseId
   - Look up phase -> session -> program -> requireOrgAdmin
   - Check no running live state
   - Delete the phase document
   - Returns: v.null()

Create **convex/course/sessionQueries.ts** with two queries:

1. **getSessionPhases** query:
   - Args: sessionId
   - Access check: look up session -> program -> checkProgramAccess (both admins and participants can read)
   - Return all phases for sessionId ordered by orderIndex
   - Returns validator: v.array with phase shape

2. **getLiveState** query:
   - Args: sessionId
   - Access check: look up session -> program -> checkProgramAccess
   - Return the sessionLiveState document for this sessionId, or null
   - Returns: v.union(liveStateValidator, v.null())

Both files must follow convex/CLAUDE.md: args + returns validators on every function, no .filter() (use .withIndex()), no default exports.
</action>
