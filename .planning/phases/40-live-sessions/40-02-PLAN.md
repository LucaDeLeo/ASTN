<task type="auto">
  <name>Task 1: Live session lifecycle mutations and presence</name>
  <files>convex/course/sessionRunner.ts</files>
  <action>
Create convex/course/sessionRunner.ts with the following mutations. All use requireOrgAdmin for auth (except updatePresence which uses requireAuth for participants). All lifecycle mutations check current state before acting (idempotency per CONTEXT.md decision).

**startSession** mutation:

- Args: sessionId: v.id('programSessions')
- Look up session -> program -> requireOrgAdmin
- Check one-live-session-per-program invariant: query sessionLiveState with by_programId_and_status index for status='running'. If found, throw ConvexError('A session is already running for this program').
- Get first phase by orderIndex (query sessionPhases by_sessionId_and_order)
- If no phases exist, throw error
- Create sessionLiveState document: status='running', currentPhaseId=first phase, phaseStartedAt=Date.now(), phaseDurationMs=first phase's durationMs, activePromptIds=first phase's promptIds or [], startedAt=Date.now(), startedBy=userId
- Returns: v.id('sessionLiveState')

**advancePhase** mutation:

- Args: sessionId: v.id('programSessions')
- Look up session -> program -> requireOrgAdmin
- Get liveState by_sessionId. If not found or status!='running', return null (idempotent)
- Record phase result: insert sessionPhaseResults with actualDurationMs = Date.now() - phaseStartedAt, startedAt=phaseStartedAt, endedAt=Date.now()
- Get all phases ordered by orderIndex, find current phase index, get next phase
- If no next phase: patch liveState status='completed', completedAt=Date.now()
- If next phase exists: patch liveState with new currentPhaseId, phaseStartedAt=Date.now(), phaseDurationMs=nextPhase.durationMs, activePromptIds=nextPhase.promptIds or []
- Returns: v.null()

**extendPhase** mutation:

- Args: sessionId: v.id('programSessions'), additionalMs: v.number()
- Look up session -> program -> requireOrgAdmin
- Get liveState. If not found or status!='running', return null (idempotent)
- Patch liveState.phaseDurationMs += additionalMs
- Returns: v.null()

**skipPhase** mutation:

- Same as advancePhase but does NOT record a phase result (phase was skipped, not completed)
- Returns: v.null()

**endSession** mutation:

- Args: sessionId: v.id('programSessions')
- Look up session -> program -> requireOrgAdmin
- Get liveState. If not found or status!='running', return null (idempotent)
- Record final phase result if current phase was started
- Patch liveState status='completed', completedAt=Date.now()
- Returns: v.null()

**createAdHocPrompt** mutation:

- Args: sessionId: v.id('programSessions'), title: v.string(), body: v.optional(v.string()), fields: same validator as coursePrompts fields
- Look up session -> program -> requireOrgAdmin
- Get liveState. Must be running.
- Get current phase
- Create a new coursePrompt with attachedTo: { type: 'session_phase', sessionId, phaseId: liveState.currentPhaseId }, revealMode: 'immediate'
- Add the new prompt's ID to liveState.activePromptIds (patch liveState)
- NOTE: This implements SESS-09 as "ad-hoc prompt appears for all participants immediately" by adding to the current phase's activePromptIds. This supersedes the original REQUIREMENTS.md wording of "new phase appears" — no new phase is created; the prompt is injected into the current phase and participants see it immediately because they subscribe to liveState.activePromptIds.
- Returns: v.id('coursePrompts')

**updatePresence** mutation (public, participant-accessible):

- Args: sessionId: v.id('programSessions'), phaseId: v.id('sessionPhases'), status: v.union(v.literal('typing'), v.literal('idle'), v.literal('submitted'))
- requireAuth (not requireOrgAdmin — participants call this)
- Check program access via checkProgramAccess
- Upsert sessionPresence: query by_sessionId_and_userId, if exists patch (phaseId, status, lastSeen=Date.now()), else insert
- Returns: v.null()

IMPORTANT: Use ConvexError (not plain Error) for user-facing errors. Follow convex/CLAUDE.md: args+returns validators, no .filter(), use .withIndex().

<task type="auto">
  <name>Task 2: Pairing algorithms and extended queries</name>
  <files>convex/course/sessionPairing.ts, convex/course/sessionQueries.ts</files>
  <action>
Create **convex/course/sessionPairing.ts** with two mutations:

**generatePairs** mutation:

- Args: sessionId: v.id('programSessions'), phaseId: v.id('sessionPhases'), strategy: v.union(v.literal('random'), v.literal('complementary'))
- Look up session -> program -> requireOrgAdmin
- Get liveState. Must be running.
- Get present participants: query sessionPresence by_sessionId, filter to lastSeen > Date.now() - 30000 (30s heartbeat window). Collect unique userIds.
- If strategy='random': use Fisher-Yates shuffle, then pair sequentially. If odd count, last 3 form a trio.
- If strategy='complementary': read the phase's pairConfig.sourcePromptId and sourceFieldId. Query coursePromptResponses for that promptId (by_promptId index), collect submitted responses for present users. Group by their selected choice on sourceFieldId. Pair across different choice groups (cross-choice for richer discussion per research recommendation). Remaining unpaired users fall back to random pairing.
- Insert sessionPairAssignments with pairs array, strategy, timestamps, createdBy.
- Returns: v.id('sessionPairAssignments')

**setManualPairs** mutation:

- Args: sessionId: v.id('programSessions'), phaseId: v.id('sessionPhases'), pairs: v.array(v.object({ members: v.array(v.string()) }))
- Look up session -> program -> requireOrgAdmin
- Get liveState. Must be running.
- Insert sessionPairAssignments with strategy='manual', provided pairs, timestamps.
- Returns: v.id('sessionPairAssignments')

Helper functions (not exported, top of file):

- `shuffle<T>(array: T[]): T[]` — Fisher-Yates shuffle
- `createPairsFromList(userIds: string[]): Array<{ members: string[] }>` — sequential pairing with trio for odd count

Extend **convex/course/sessionQueries.ts** with four additional queries:

**getPresence** query:

- Args: sessionId: v.id('programSessions')
- requireOrgAdmin (facilitator-only view)
- Query sessionPresence by_sessionId, filter to lastSeen > Date.now() - 30000
- Return array of { userId, phaseId, status, lastSeen }

**getPairAssignments** query:

- Args: sessionId: v.id('programSessions'), phaseId: v.optional(v.id('sessionPhases'))
- checkProgramAccess (both admins and participants)
- If phaseId provided, query by_sessionId_and_phaseId. Otherwise query by_sessionId.
- Return pair assignments

**getPhaseResults** query:

- Args: sessionId: v.id('programSessions')
- checkProgramAccess
- Query sessionPhaseResults by_sessionId
- Return array of results

**getMyPairs** query:

- Args: sessionId: v.id('programSessions'), phaseId: v.id('sessionPhases')
- requireAuth (participant query)
- Query sessionPairAssignments by_sessionId_and_phaseId
- Find the pair group containing the current userId
- Return the members array (other participants in your pair/trio), or null if not paired

NOTE on presence filtering: Convex does not allow .filter() — but since we need to filter by lastSeen > threshold (a computed value), and there's no index for range on lastSeen, use .collect() on the by_sessionId index and then filter in application code. This is acceptable for the scale (~10-50 participants per session). Add a code comment explaining why.

Follow convex/CLAUDE.md for all functions.
