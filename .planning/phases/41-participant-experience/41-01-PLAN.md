  <action>
Update the material object in `convex/schema.ts` (inside the `programModules` table definition, around line 1065-1077) to add three new optional fields and one new type literal:

1. Change `url: v.string()` to `url: v.optional(v.string())` -- audio materials have no external URL
2. Add `v.literal('audio')` to the type union (alongside link, pdf, video, reading)
3. Add `isEssential: v.optional(v.boolean())` -- undefined means essential (backward compat with all existing materials)
4. Add `storageId: v.optional(v.id('_storage'))` -- for audio file references

Update the `materialValidator` in `convex/programs.ts` (around line 590-600) to match the schema changes exactly:

- `url: v.optional(v.string())`
- Add `v.literal('audio')` to the type union
- Add `isEssential: v.optional(v.boolean())`
- Add `storageId: v.optional(v.id('_storage'))`

Update the `MaterialItem` interface in `src/lib/program-constants.ts` (around line 26-31):

- Change `url: string` to `url?: string`
- Add `'audio'` to the type union
- Add `isEssential?: boolean`
- Add `storageId?: string` (frontend receives string, not Id type)
- Add `audioUrl?: string` (resolved URL from storage, populated by backend query)

IMPORTANT: The `materialValidator` in programs.ts is used as both an arg validator for createModule/updateModule AND as a return validator in getParticipantProgramView. Create a separate `materialReturnValidator` that extends materialValidator with `audioUrl: v.optional(v.string())` and use it in the return type of getParticipantProgramView. Keep the arg validator (`materialValidator`) without `audioUrl` since that field is computed server-side, not sent by the client.

<task type="auto">
  <name>Task 2: Backend query and mutation updates</name>
  <files>convex/programs.ts</files>
  <action>
Three changes to `convex/programs.ts`:

**A. Audio URL resolution in getParticipantProgramView (around line 1599-1608):**
Update the `modules` mapping in the return block to resolve audio URLs inline. For each module's materials array, use `Promise.all` to resolve `storageId` to `audioUrl` via `ctx.storage.getUrl(storageId)`. This follows the existing inline URL resolution pattern used for org logos (see `convex/events/queries.ts:96`).

Change:

```
modules: modules.map((m) => ({ ... materials: m.materials, ... }))
```

To:

```
modules: await Promise.all(modules.map(async (m) => ({
  ...existing fields...,
  materials: m.materials
    ? await Promise.all(m.materials.map(async (mat) => ({
        ...mat,
        audioUrl: mat.storageId ? await ctx.storage.getUrl(mat.storageId) : undefined,
      })))
    : undefined,
  ...existing fields...,
})))
```

Update the `modules` return validator to use `materialReturnValidator` (from Task 1) instead of `materialValidator`.

**B. Blob cleanup in updateModule and deleteModule mutations:**
In `updateModule`: Before patching, compare old materials array with new materials array. For any `storageId` present in old materials but NOT in new materials, call `await ctx.storage.delete(oldStorageId)`. This prevents orphaned blobs when audio materials are removed or replaced.

In `deleteModule` (if it exists -- check): When deleting a module, iterate its materials and delete any storageIds via `ctx.storage.delete()`.

**C. Add promptCompletionByModule to getParticipantProgramView:**
After fetching `myMaterialProgress`, add a query for prompt completion per module. For each module:

1. Query `coursePrompts` with index `by_module` where `moduleId` matches
2. For each prompt found, check if the user has a submitted response in `coursePromptResponses` (use index `by_prompt_and_user`)
3. Return `promptCompletionByModule` as an array of `{ moduleId, totalPrompts, completedPrompts }` objects

Add the corresponding return validator field:

```
promptCompletionByModule: v.array(v.object({
  moduleId: v.id('programModules'),
  totalPrompts: v.number(),
  completedPrompts: v.number(),
}))
```

Also update the `materialValidator` validation in `handleSubmit` of createModule to accept materials with EITHER a url OR a storageId: change the filter from `m.label.trim() && m.url.trim()` to `m.label.trim() && (m.url?.trim() || m.storageId)` -- but this validation is client-side in ModuleFormDialog (Plan 02), so for the backend: ensure the createModule/updateModule mutations don't reject materials that have storageId but no url. The current arg validator with `url: v.optional(v.string())` already handles this.
</action>
