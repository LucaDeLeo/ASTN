**MaterialIcon.tsx:**

**ModuleFormDialog.tsx -- Essential/Optional toggle:**
For each material row in the form (inside the `materials.map` loop):

1. Add a toggle/switch for essential vs optional. Use a simple button or the existing shadcn `Switch` component (check if available, otherwise use a checkbox). Position it at the bottom of each material card, labeled "Essential" with a toggle. When `isEssential` is `false`, show "Optional" label in muted text.
2. Default new materials to `isEssential: undefined` (treated as essential).
3. Wire `updateMaterial(i, 'isEssential', !mat.isEssential)` to toggle.
4. Update the `addMaterial` function: new materials default shape should include the new fields: `{ label: '', url: '', type: 'link' as const, isEssential: undefined }`.

**ModuleFormDialog.tsx -- Audio type in material type picker:**

1. Add `<SelectItem value="audio">Audio</SelectItem>` to the material type `<Select>` dropdown (after "Reading").
2. When material type is `'audio'`, conditionally render an audio upload UI instead of the URL input:
   - Show a file input (`<input type="file" accept="audio/*">`) styled as a button labeled "Upload Audio"
   - On file selection: validate MIME type (`file.type.startsWith('audio/')`) and size (max 50MB), then call `generateUploadUrl` mutation, POST the file to the upload URL, extract `storageId` from the response JSON, and call `updateMaterial(i, 'storageId', storageId)`. Also set `updateMaterial(i, 'url', undefined)`.
   - While uploading, show a Spinner and disable the button
   - After upload, show the filename or "Audio uploaded" confirmation with option to re-upload
   - Import `useMutation` for `api.upload.generateUploadUrl` (already exists in `convex/upload.ts`)
3. When material type is NOT `'audio'`, show the existing URL input as before.
4. If the user switches type FROM audio to something else, clear the `storageId` field: `updateMaterial(i, 'storageId', undefined)`.

**ModuleFormDialog.tsx -- Validation update:**
Update the `validMaterials` filter in `handleSubmit` (line 96-98) from:

```
materials.filter((m) => m.label.trim() && m.url.trim())
```

To:

```
materials.filter((m) => m.label.trim() && (m.url?.trim() || m.storageId))
```

This ensures audio materials (which have `storageId` but no `url`) pass validation.

**ModuleFormDialog.tsx -- Props type update:**
Update the `module` prop type to include the new MaterialItem fields. The `MaterialItem` import already covers this since Plan 01 updated the interface.
